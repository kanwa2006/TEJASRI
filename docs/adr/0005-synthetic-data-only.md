# ADR 0005 — Synthetic patient data only (Synthea)

- **Status:** Accepted
- **Date:** 2026-07-13

## Context

TEJASRI handles PHI-shaped data (conditions, medications, clinical notes). Real PHI would impose HIPAA-grade obligations incompatible with an open-source project and free-tier LLM APIs (whose inputs may be used for training).

## Decision

All patient data in every environment (dev, CI, demo, docs, tests) is **synthetic**, generated by [Synthea](https://synthetichealth.github.io/synthea/). Real patient data is prohibited by policy, documented in CONTRIBUTING.md, SECURITY.md, and enforced in review.

## Rationale

- Synthea produces realistic FHIR/CSV records "free from cost, privacy, and security restrictions."
- Removes privacy risk from free-tier LLM usage and from public demo deployments.
- Keeps the platform honest: production-grade *engineering* without production *liability*.

## Consequences

- Ingest pipeline targets Synthea CSV/FHIR output; a future FHIR adapter (roadmap) would be the path to real-world integration under proper compliance.
- Every demo and screenshot must be reproducible from a seeded synthetic dataset.
- The platform is still engineered as if data were real (RLS, encryption in transit, audit logs) — synthetic data is a policy choice, not an excuse.
